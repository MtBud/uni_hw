Tato semestrální práce patří do kategorie konzolových utilit. Vaším cílem je vytvořit nástroj, který uživatel z terminálu spustí (s různými parametry). Případné parametry nástroje se nesnažte definovat přímo v kódu (a to ani pomocí konstant). Umístěte je do konfiguračních souborů (jednoho nebo více), které váš program bude načítat.

Úkolem je naprogramovat jednoduchý HTTP server s podporou různých generátorů obsahu. Alternativně můžete implementovat i jiný známý protokol, který se vzhledem k požadavkům hodí (př. FTP server, …). Pokud si nejste jisti, diskutujte nejprve s pověřenou osobou pro semestrální práce.

Nástroj musí implementovat:

  1. Poskytování obsahu následující druhu:

    1. statický obsah (konkrétní soubor),

    2. výpis obsahu adresáře (pokud je v nastavení povoleno),

    3. spuštění skriptu (př. sh, c, c++, …),

    4. zaslání signálu pro ukončení serveru,

    5. neexistující stránka.

  2. Server musí umožnit:

    1. získávání statických souborů pomocí GET metody,

    2. ukládání statických souborů pomocí POST metody.

  3. Nástroj musí být konfigurovatelný ze souboru:

    1. parametry sítě (alespoň IP adresa, port, dále příp. doména, …),

    2. omezení práce s konkrétními adresáři (konfigurace obsahuje na jaké adresy se mapují jednotlivé adresáře),

    3. log přenosů (jednotlivé požadavky a zkrácené odpovědi),

    4. definice přípon pro zpracování pomocí skriptu (př. .c, .cpp pomocí g++ a následně ./a.out, .sh pomocí sh …),

    5. na jakou adresu (příp. metodu) se mapuje vypnutí serveru.

Kde lze využít polymorfismus? (doporučené)

  • druh zpracování souboru: adresář, statický soubor (HTML, obrázek, …), skript, nepodporovaný formát, …

  • různé chování metod pro zpracování obsahu: GET, POST, PUT, DELETE, …

  • logování: na konzoli, do souboru, do databáze, …

  • styl logování: jednoduchá hlavička, kompletní požadavek (např. pouze pro chybové stavy), statistika přístupů, …

Další informace:

  • Rozšířením může být například podpora HTTPS nebo rozšíření o vícevláknové zpracování. V takovém případě bude vhodné přidat možnost zapnout a vypnout danou funkcionalitu pomocí vhodného přepínače.
